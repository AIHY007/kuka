# 售后工程师技能图鉴系统 UI 升级方案

## 1. UI 整体升级方案

目标定位：企业级 3A 游戏角色图鉴系统 UI。视觉语言来自角色档案、能力认证徽章、成就统计面板和数据排行榜，但整体控制在企业正式场景可接受的克制范围内。

页面结构：
- 首页采用三栏信息架构：左侧筛选矩阵、中部工程师角色卡网格、右侧勋章排行榜。
- 顶部系统栏承担产品识别、全局搜索和排行榜快速入口。
- 个人页采用角色档案详情页结构：主档案面板、成就总览、勋章分类展示。

设计系统：
- 背景：深蓝灰与炭黑科技底，避免纯黑。
- 主强调：冷蓝用于系统行动与信息聚焦。
- 稀有度：传奇金、史诗紫、稀有蓝、普通银灰。
- 边框：1px 微发光渐变描边。
- 阴影：低透明度多层阴影，强调层级但不制造强烈炫光。
- 圆角：8px、12px、14px 三档。

## 2. GitHub 代码修改建议

本次已直接完成的前端结构升级：
- `index.html` 变为轻量入口，只保留页面骨架。
- `styles.css` 承载视觉系统、响应式布局、角色卡、勋章、排行榜、档案页样式。
- `app.js` 承载数据读取、兼容层、组件渲染、筛选、排行榜、个人页切换。
- `data.json` 新增结构化字段，同时保留旧字段，保证当前系统继续兼容。

建议后续维护规则：
- 修改人员、勋章、培训、证书数据时只改 `data.json`。
- 新增 UI 组件时优先在 `app.js` 的 `Components` 对象中新增渲染函数。
- 新增视觉主题时只改 `styles.css` 的 `:root` 变量和稀有度变量。

## 5. Canva 视觉设计方向

已触发 Canva 设计准备：
- 企业风宣传海报：主题为内部能力认证体系发布，深蓝灰科技底，角色图鉴式主视觉。
- 勋章排行榜展示卡片：主题为能力认证数据榜单，突出前三名、总勋章、传奇勋章、技能勋章。

海报方向：
- 标题：售后工程师技能图鉴系统。
- 副标题：能力认证、技能资产、团队成长可视化。
- 主视觉：抽象档案卡片 + 工业奖章矩阵。
- 色彩：深蓝灰底、冷蓝线框、少量金色强调。

排行榜卡片方向：
- 三榜单并列：总勋章、传奇勋章、技能勋章。
- 前三名使用金色序号，其余使用冷蓝/银灰。
- 背景使用微网格与弱发光面板，不使用卡通图形。

## 6. Remotion 动效脚本方案

Remotion 规则：动效使用 `useCurrentFrame()`、`useVideoConfig()`、`interpolate()` 与 `Easing.bezier()`，不使用 CSS 动画。

首页加载动画：
```tsx
const titleIn = interpolate(frame, [0.15 * fps, 0.75 * fps], [0, 1], {
  easing: Easing.bezier(0.16, 1, 0.3, 1),
  extrapolateLeft: "clamp",
  extrapolateRight: "clamp"
});

const cardProgress = (index: number) => interpolate(
  frame,
  [(0.45 + index * 0.06) * fps, (0.95 + index * 0.06) * fps],
  [0, 1],
  {
    easing: Easing.bezier(0.16, 1, 0.3, 1),
    extrapolateLeft: "clamp",
    extrapolateRight: "clamp"
  }
);
```

勋章获得动画：
```tsx
const focus = interpolate(frame, [0, 0.5 * fps], [0, 1], {
  easing: Easing.bezier(0.22, 1, 0.36, 1),
  extrapolateLeft: "clamp",
  extrapolateRight: "clamp"
});

const medalScale = interpolate(focus, [0, 1], [0.86, 1]);
const glow = interpolate(frame, [0.35 * fps, 1.2 * fps], [0, 1], {
  easing: Easing.bezier(0.45, 0, 0.55, 1),
  extrapolateLeft: "clamp",
  extrapolateRight: "clamp"
});
```

个人档案进入动画：
```tsx
const panel = interpolate(frame, [0.1 * fps, 0.72 * fps], [0, 1], {
  easing: Easing.bezier(0.16, 1, 0.3, 1),
  extrapolateLeft: "clamp",
  extrapolateRight: "clamp"
});

const countValue = Math.round(interpolate(frame, [0.45 * fps, 1.2 * fps], [0, targetCount], {
  easing: Easing.bezier(0.16, 1, 0.3, 1),
  extrapolateLeft: "clamp",
  extrapolateRight: "clamp"
}));
```

## 7. Hyperframes 应用点说明

当前前端已实现可落地的轻动态角色状态：
- 头像状态环使用极轻微呼吸光效。
- 角色卡 hover 时边框发光增强，并有一次性金属扫光。
- 传奇勋章使用非常克制的微光脉冲。
- 个人档案头像延续状态能量环，形成从大厅到档案的动效连续性。

Hyperframes 视频化建议：
- 将 `styles.css` 的视觉变量整理成 `DESIGN.md`。
- 建立 3 个场景：系统启动、角色档案展开、勋章认证浮现。
- 使用 GSAP `from()` 做入口，避免跳切。
- 多场景之间使用柔和 crossfade 或线性遮罩 reveal，不使用强烈 glitch。
